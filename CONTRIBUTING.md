# Contributing to BORK Community

We welcome contributions from everyone. This guide explains how to get involved.

## Ways to Contribute

### Documentation
- Fix typos or unclear explanations
- Add missing information to existing guides
- Create new tutorials or guides
- Improve examples

### Skills
- Share skills you've built
- Improve existing skills
- Add examples showing skill usage

### Hooks and Agents
- Share useful hook configurations
- Contribute custom agent examples
- Document real-world use cases

### Bug Reports
- Report broken links or outdated info
- Flag security concerns
- Note compatibility issues

## Getting Started

1. Fork the repository
2. Create a branch for your changes
3. Make your changes
4. Submit a pull request

## Pull Request Guidelines

### Before Submitting

- [ ] Test any code examples you've added
- [ ] Verify all links work
- [ ] Check spelling and grammar
- [ ] Follow existing formatting conventions

### PR Description

Include:
- What you changed and why
- Any testing you did
- Screenshots if relevant (for UI/formatting changes)

### Commit Messages

Keep them clear and descriptive:
```
Add Python CLAUDE.md template
Fix broken link in getting-started guide
Improve explanation of plan mode
```

## Content Guidelines

### Writing Style

- **Lead with value**: Start with what the reader can do
- **Be specific**: Use concrete examples, not vague descriptions
- **Stay accessible**: Explain jargon, link to prerequisites
- **Keep it scannable**: Short paragraphs, bullet points, headers

### Code Examples

- Test all code before submitting
- Include comments for non-obvious parts
- Show complete, runnable examples when possible

### Skills Format

When adding a skill:

```
05-skills/
└── your-skill-name/
    ├── README.md       # Description, usage, examples
    ├── SKILL.md        # The actual skill file
    └── examples/       # Optional usage examples
```

The README should include:
- What the skill does
- How to install it
- Basic usage examples
- Any prerequisites

## Review Process

1. Maintainers review PRs within a few days
2. We may request changes or clarifications
3. Once approved, we'll merge your contribution
4. Contributors get credited in HALL_OF_FAME.md

## Community Guidelines

- Be helpful and constructive
- Respect different skill levels
- Focus on improving the resource
- No spam or self-promotion

## Questions?
